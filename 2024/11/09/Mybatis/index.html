Mybatis

什么是Mybatis

  • Mybatis 是一个持久层框架
  • MyBatis 避免了几乎所有的 JDBC 代码和手动设置参数以及获取结果集的过程，减少了代码的冗余，减少程序员的操作。
  • MyBatis 可以使用简单的 XML 或注解来配置和映射原生信息，将接口和 Java 的实体类映射成数据库中的记录。
  • MyBatis 支持定制化 SQL、存储过程以及高级映射，可以在实体类和 SQL 语句之间建立映射关系，是一种半自动化的 ORM 实现。

模型对象

  • 值对象：实体对象中的一个值，有一定的范围或者特定的几个取值，那么可以定义一个更贴近场景业务的对象。（类似VO）
  • 实体对象：对数据库对象的抽象，大多数时候是 1:1 的关系结构，在一些复杂的模型场景中会是1:n的结构。
  • 聚合对象：聚合对象是对实体对象和值对象的封装，代表着一类业务的聚合。通常是作为 service 服务层中入参出现。（类似BO业务对象，包含多个PO或VO对象的属性，以及这些属性的业务逻辑方法。）

XML配置

XML标签，配置各种SQL语句

  • <mapper>：所有SQL映射语句都要在元素内，包含一个namespace属性，这个namespace属性不妨直接定义为对应的DAO类，保证唯一
  • <resultMap>：用于将数据库查询结果映射到Java对象的属性上。MyBatis通过这个标签来定义结果集和Java对象之间的关系，从而实现数据库字段和Java属性的对应。例如employee_number列会映射到com.example.Employee类中的employeeNumber字段
    • 例子：
      1. <collection>：用于映射集合属性（一般是一对多关系），比如对象的集合属性或列表属性。比方说当需要用join连接表的时候，会出现一对多，那么就可以用这个标签。
  • <select>：要检索数据，使用“SELECT”映射器语句。
  • <insert>：要将值插入表中，我们必须配置插入映射查询。
    • 属性：
      1. id：语句的唯一标识
      2. parametertype：是将被传递到语句中的类名或参数的别名。
    • 多个入参？
      1. XML定义：在没有 @Param 注解的情况下，可以直接使用 param1、param2 等方式来引用参数。
      2. 使用@Param注解：
        • 通过参数占位符#{}使用入参（#{}是预编译参数传递，没有注入风险。${}是字符串替换，存在SQL注入风险）
  • <update>：要使用MyBatis更新现有记录的值，需要配置映射查询更新。更新映射查询的属性与插入映射查询的属性相同，同样是id和parametertype。
  • <delete>：要使用MyBatis删除现有记录的值，需要配置映射查询“delete”。“delete”映射查询的属性与插入和更新映射查询相同。
  • <foreach>：用于在 SQL 语句中循环遍历集合（如 List 或 Map），并生成多个参数值或 SQL 片段。它在批量插入、批量更新、构建 IN 查询条件等场景中非常常用。

    • 属性：
      1. collection：定需要遍历的集合，可以是 List、Map、数组等。
      2. item：定义每次遍历的元素名称。假设设置为 “item”，表示集合中的每个元素会被命名为 item，供循环体内部引用。
      3. separator：指定循环生成的 SQL 片段之间的分隔符。这里的 separator=”,”，表示每次循环生成的 (#{item.employeeNumber}, #{item.employeeName}, …) 会以逗号分隔。
  • <if>：条件标签，用于动态SQL。属性是test，属性值是条件
    • 例子：
  • <where>：多个if标签的情况下减少判断，自动处理AND\OR避免SQL语法错误
    • 例子：if 语句中判断条件为 true 时，where 关键字才会加入到组装的 SQL 里面，否则就不加入。where 会检索语句，它会将 where 后的第一个 SQL 条件语句的 AND 或者 OR 关键词去掉。
  • <set>：update 语句可以使用 set 标签动态更新列。set 标签可以为 SQL 语句动态的添加 set 关键字，剔除追加到条件末尾多余的逗号。
  • <bind>：每个数据库的拼接函数或连接符号都不同，例如 MySQL 的 concat 函数、Oracle 的连接符号“||”等。这样 SQL 映射文件就需要根据不同的数据库提供不同的实现，显然比较麻烦，且不利于代码的移植。幸运的是，MyBatis 提供了 bind 标签来解决这一问题。bind 标签可以通过 OGNL 表达式自定义一个上下文变量。
    • 属性：
      1. value：对应传入实体类的某个字段，可以进行字符串拼接等特殊处理。
      2. name：给对应参数取的别名。
