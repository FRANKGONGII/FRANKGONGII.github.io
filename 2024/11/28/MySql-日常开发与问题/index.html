MySql-日常开发与问题

常见问题

连接不上数据库

  1. 账号密码数据库名
  2. 检查用的那个application.yml
  3. 时区，ssl设置：serverTimezone=UTC&useSSL=false，连不上可以考虑关闭ssl
  • 本地可以使用的配置

TimeZone问题

  • 运行正确的插入语句： 运行错误的插入语句： 可以发现由程序透传的#{createTime}和now()相差了8个小时
  • 问题解决：
    1. 检查JDBC：不要使用8.0.22，使用之后的版本，如23
    2. 连接参数配置：&serverTimezone=Asia/Shanghai
    3. 设置MYSQL的参数配置：
    • SET time_zone = 'SYSTEM';
    • SET time_zone = '+8:00';
    1. 修改服务器（本地）时间：sudo timedatectl set-timezone Asia/Shanghai
